# ステータス API

## GET /healthcheck/status

### 概要

Akashic システムの現在のステータスを返します。システムが正常に稼働している場合は、成功を示す HTTP status code となります。

### 特記事項

この API は [GET /healthcheck](./healthcheck.md#get-healthcheck) と同じ動作仕様ですが、以下の点が異なります。

1. [GET /healthcheck](./healthcheck.md#get-healthcheck)
   1. サーバ管理者向けの情報です。システム利用者はこちらを参照する必要はありません。
   1. 「サーバの健全性」を表します。サーバが動作している場合に正常を返します。
1. **GET /healthcheck/status** （当該 API）
   1. システム利用者向けの情報です。**システムの利用者が状態を参照する場合は、こちらの API をご利用ください。**
   1. 「システムの状態」を表します。 システムが動作している場合に正常を返します。
   1. 「サーバは動作しているが、システムの復元中など利用者にとって正常ではない」ケースが存在します。

成功以外を返す一例:

- **システムが稼働していない**
  - 多くの場合は `HTTP status code: 404` を返します。
  - メンテナンス時間外にこの状態になるとき、Akashic システムそのものがダウンしている可能性があります。
- **一時的にレスポンスを返せない**
  - 多くの場合は `HTTP status code: 504` を返します。
  - API, DNS, LB, Proxy などの一時的な負荷によってこの状態になることがあります。
  - リトライによって成功することがあります。リトライ回数についてはそれ自体が負荷にならないようご注意ください。

### リクエスト

この API のエンドポイントはバージョン文字列 (`/v1.0/` など）を含めません。

```sh
curl ${ENDPOINT}/healthcheck/status
```

### レスポンス

この API はレスポンス Body に有意な値を返しません。 HTTP status code のみを取得してください。

```sh
curl ${ENDPOINT}/healthcheck/status -o /dev/null -w '%{http_code}\n' -s
200
```

| ステータスコード | エラーコード | 備考          |
| ---------------- | ------------ | ------------- |
| 200              | 成功         |               |
| 200 以外         | 失敗         | 404, 504 など |
