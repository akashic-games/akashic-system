## Database Migrations

最近だと「クラウドサービスのデータベースサービスに移行すること」もデータベースマイグレーションというらしいですが、それではなく、スキーママイグレーションとも呼ばれるほうのやつです。
[Active Record のマイグレーション](https://railsguides.jp/active_record_migrations.html)あたりを読めばわかると思います。

## ここで管理するものと、別管理のもの

データベースマイグレーションと言われるくらいなので、ここで管理するスキーマは、データベースのスキーマに限ります。
S3 のようなオブジェクトストレージ、RabbitMQ のようなメッセージキュー、Redis のような KVS などは、ここではなく、[scripts ディレクトリ](../scripts)にあるそれっぽい名前のスクリプトで管理されています。

データベーススキーマは中のデータやインデックスなどを含めて作ったり消したりをします。
しかし、オブジェクトストレージのバケット や AMQP のキューやエクスチェンジ などは「存在していれば良い」だけなので、わざわざデータベースマイグレーションほどの高機能なツールを使う必要がありません。
まぁ、使う必要が出てきても、一般的なマイグレーションツールでは対応していないバックエンドだと思うので、どっちにしろ自力で管理する必要があるんですけどね。
